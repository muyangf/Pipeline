# 📘 **offline-default.md（离线默认预设 · 最终版）**

**—— 离线模型 · 本地推理 · 稳定输出 · 中文优先**

# # 1. 适用范围（Scope）

本预设适用于所有 **离线模型（Offline LLM）**：

- Qwen（本地版）
- DeepSeek（本地版）
- Llama / Llama3
- Mistral
- ChatGLM
- GGUF 模型（llama.cpp / LM Studio / Ollama）

适用于以下环境：

- 无网络
- 无外部 API
- 本地推理
- 上下文窗口有限
- 输出稳定性要求高

# # 2. 中文优先（Chinese First）

离线模型在中文环境下表现更稳定，因此：

- 所有提示词必须中文
- 所有注释必须中文
- 所有文档必须中文
- 所有测试描述必须中文
- 所有审查意见必须中文

# # 3. 输出稳定性规范（Stability Standard）

离线模型容易出现：

- 输出结构不稳定
- 忘记格式
- 忘记上下文
- 生成乱码

因此必须遵循：

## 3.1 明确结构（必须）

所有输出必须包含：

- 标题
- 分节
- 列表
- 代码块

## 3.2 不得使用复杂 Markdown

避免：

- 嵌套表格
- 复杂 HTML
- 多层嵌套列表

## 3.3 不得生成过长内容

离线模型容易丢失上下文，因此：

- 每段内容不超过 20 行
- 每个代码块不超过 200 行
- 每次输出不超过 3000 字

# # 4. 离线模型提示词规范（Prompt Standard）

## 4.1 必须明确任务

例如：

Code

```
请生成中文需求文档，包含：背景、目标、功能需求、非功能需求。
```

## 4.2 必须明确格式

例如：

Code

```
请按以下结构输出：
1. 背景
2. 目标
3. 功能需求
4. 非功能需求
```

## 4.3 必须避免模糊指令

不得使用：

- “随便写点”
- “你觉得应该怎么做”
- “自由发挥”

# # 5. 离线模型代码生成规范（Code Generation Standard）

离线模型容易：

- 忘记注释
- 忘记结构
- 忘记命名规范

因此必须遵循：

## 5.1 所有代码必须包含中文注释

包括：

- 文件头注释
- 函数注释
- 行内注释

## 5.2 所有代码必须遵循智能 code-style

即：

- 自动识别业务代码
- 自动识别 AI 编排代码
- 自动切换架构规范

## 5.3 所有代码必须包含测试

测试必须中文描述。

# # 6. 离线模型的上下文管理（Context Management）

离线模型上下文窗口有限，因此：

## 6.1 必须分步骤执行

例如：

- 先生成需求
- 再生成方案
- 再生成任务
- 再生成代码

不得一次性生成所有内容。

## 6.2 必须减少重复上下文

避免：

- 重复贴大量代码
- 重复贴大量文档

## 6.3 必须使用“摘要”代替全文

例如：

Code

```
以下是前文摘要：
- 功能：搜索服务
- 输入：关键词
- 输出：搜索结果
```

# # 7. 离线模型的错误处理（Error Handling）

离线模型容易：

- 误解任务
- 忘记上下文
- 输出错误格式

因此必须：

## 7.1 明确错误提示

例如：

Code

```
如果你无法理解任务，请输出：
“我需要更多信息，请提供：xxx”
```

## 7.2 不得自行编造信息

例如：

- 不得编造 API
- 不得编造数据库结构
- 不得编造文件路径
